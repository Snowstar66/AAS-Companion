\# M3-STORY-007



\## Title

Show readiness and staffing gaps for required governance roles



\## Story Type

UI Feature



\## Value Intent

Make role readiness visible so teams can see whether the current authority model is sufficient for the selected AI level and current phase.



\## Summary

Create a readiness view that shows which required governance roles are staffed, missing or risky for the selected AI acceleration level and current initiative, and make that readiness visible from initiative or workspace context.



\## Acceptance Criteria

\- readiness view shows selected AI level

\- readiness view shows required roles for that level

\- readiness view shows assigned named people

\- missing roles are clearly marked

\- risky role combinations are clearly marked

\- readiness result can be viewed per initiative or workspace

\- readiness can influence progression visibility, but not silently approve progression

\- readiness can be resolved from initiative context even when accessed from Outcome or Story workspace

\- readiness reflects actual assigned roles, not assumed roles



\## AI Usage Scope

\- CODE

\- TEST



\## Test Definition

\- unit

\- integration

\- e2e smoke



\## Definition of Done

\- readiness view is understandable

\- missing role scenarios are easy to spot

\- staffing gaps can be demonstrated with demo data

\- risky combinations are visible without confusion

\- initiative-context readiness remains understandable from workspace entry points



\## Scope In

\- readiness UI

\- role coverage summary

\- missing/risky indicators

\- initiative-level view

\- workspace-linked readiness access



\## Scope Out

\- automatic staffing recommendations

\- HR integration

\- staffing optimization



\## Constraints

\- readiness must reflect actual assigned roles, not assumed roles

\- risky combinations must remain explicit

\- readiness should support decision-making, not hide problems

\- readiness visibility must remain separate from final tollgate approval decisions



\## Required Evidence

\- readiness screenshot

\- missing-role example

\- risky-combination example

\- workspace-linked readiness example
